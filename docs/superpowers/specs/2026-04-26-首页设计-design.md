## 1. 页面概述

| 项目 | 内容 |
|-----|------|
| **页面定位** | 业务功能核心入口，展示所有设备状态和关键指标 |
| **核心功能** | 设备状态概览、KPI指标展示、实时监控、维保状态、导航跳转 |

| **后置结果** | 可跳转至数据、记录、故障日志、维保计划等页面 |

## 3. 组件设计

### 3.1 标题栏

| 属性 | 设计规范 |
|-----|---------|
| **背景** | 橙色 #FF6600 |
| **文字** | "双城牧场-一期奶厅-一号转盘"，白色，居左 |
| **右侧** | 下拉箭头 ▼，点击弹出奶厅切换菜单 |
| **状态栏** | 白色图标，与橙色背景对比 |

### 3.2 报警条

| 状态 | 样式 | 点击行为 |
|-----|------|---------|
| 正常 | 绿色对勾 + "无异常报警"，白色卡片背景 | 无点击效果 |
| 异常 | 红色感叹号 + 异常描述，点击跳转故障日志 | 点击跳转到故障日志页面（`/fault-log`） |

**异常描述内容来源**：
- 来源：首页数据 `alarm.description` 字段
- 如果 `description` 为空，显示默认文案"设备异常"
- 异常描述由后端实时推送，前端仅做展示

### 3.3 指标卡片（2x2矩阵）

| 序号 | 标题 | 示例值 | 单位 |
|-----|------|-------|------|
| 1 | 识别牛数 | 258 | 头 |
| 2 | 乳头识别率 | 99.8 | % |
| 3 | 药液用量 | 1250 | L |
| 4 | 药液均量 | 4.8 | ML |

**卡片设计**：

| 属性 | 设计规范 |
|-----|---------|
| **布局** | 上：图标+标题 / 下：数值+单位 |
| **图标** | 对应功能图标（牛/图表/水滴），橙色小圆形背景 |
| **标题** | 灰色，较小字号 |
| **数值** | 橙色，加粗，大字号 |
| **单位** | 橙色，普通字号 |
| **背景** | 白色卡片，圆角8px |
| **间距** | 卡片间距12dp |

## 6. 数据结构

### 首页数据模型

```typescript
interface HomePageData {
  // 位置信息
  location: {
    farmName: string;      // "双城牧场"
    period: string;       // "一期奶厅"
    deviceName: string;   // "一号转盘"
  };

  // 报警状态
  alarm: {
    hasAlarm: boolean;
    description?: string;
  };

  // KPI指标
  kpi: {
    cowCount: number;        // 识别牛数
    recognitionRate: number; // 乳头识别率 %
    totalUsage: number;      // 药液用量 L
    avgUsage: number;        // 药液均量 ML
  };

  // 监控信息
  monitors: [
    { type: "监控视角", status: "运行中", thumbnailUrl: string },
    { type: "作业视角", status: "运行中", thumbnailUrl: string }
  ];

  // 设备状态
  deviceStatus: Array<{
    name: string;
    status: "normal" | "warning" | "error";
  }>;

  // 维保状态
  maintenanceStatus: Array<{
    name: string;
    status: "normal" | "warning" | "error";
    expireTime?: string;
  }>;
}
```

## 10. 备注

- "上总览、中监控、下细节"的布局逻辑符合工业监控场景操作习惯
