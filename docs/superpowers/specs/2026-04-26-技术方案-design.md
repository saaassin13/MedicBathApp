## 2. 架构设计

### 整体架构：Clean Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    Presentation Layer                    │
│   (UI Widgets, Pages, Providers/State Management)        │
├─────────────────────────────────────────────────────────┤
│                    Application Layer                    │
│   (Use Cases, DTOs, Mappers)                            │
├─────────────────────────────────────────────────────────┤
│                      Domain Layer                       │
│   (Entities, Repository Interfaces, Use Case Interfaces)│
├─────────────────────────────────────────────────────────┤
│                       Data Layer                        │
│   (Repository Impl, Data Sources, API Client, Mock)      │
└─────────────────────────────────────────────────────────┘
```

### 分层职责

| 层级 | 职责 | 示例 |
|-----|------|-----|
| **Presentation** | UI 展示、状态管理、用户交互 | Page, Widget, Notifier/Provider |
| **Application** | 用例编排、数据转换（**当前简化实现暂不启用**） | UseCase, DTO, Mapper |
| **Domain** | 业务实体、仓库接口、用例接口 | Entity, Repository Interface |
| **Data** | 接口实现、数据来源、API调用 | RepositoryImpl, DataSource, Mock |

> **架构简化说明**：考虑到当前阶段以 Mock 数据为主，原定的 Application 层（UseCase）在实际实现中暂不使用。所有业务逻辑暂时直接在 Presentation 层通过 Provider 调用 Repository 获取数据。这样的简化可以加快前期开发速度，后续接入真实 API 时可按需补充 UseCase 层。

## 4. 数据流设计

```
用户操作 → Page Widget → Provider(Notifier) → Repository(Interface)
                                                   ↓
                                             DataSource (Mock/API)
```

> **当前简化数据流**：省略了 UseCase 层，Provider 直接调用 Repository。

### 关键原则

- Domain 层只定义 Interface，不实现
- Data 层实现 Repository 和 DataSource
- Presentation 层通过 Riverpod Provider 注入依赖
- Mock 数据通过 RepositoryImpl 切换，无需修改上层代码

## 8. 路由配置

### 路由结构总览

| 分类 | 路由路径 | 页面 | 导航特性 |
|-----|---------|------|---------|
| 认证流程 | `/login` | 登录页面 | 入口，无前置页面 |
| 认证流程 | `/select-farm` | 选择奶厅页面 | 登录后进入，支持返回登录 |
| Tab页面 | `/home` | 首页 | 底部Tab入口 |
| Tab页面 | `/data` | 数据页面 | 底部Tab入口 |
| Tab页面 | `/records` | 记录页面 | 底部Tab入口 |
| Tab页面 | `/personal-center` | 我的页面 | 底部Tab入口 |
| 二级页面 | `/fault-log` | 故障日志 | 全屏，无底部Tab，进入来源：首页异常点击、我的页面 |
| 二级页面 | `/maintenance-plan` | 维保计划 | 全屏，无底部Tab，进入来源：我的页面 |
| 二级页面 | `/maintenance-record` | 维保记录 | 全屏，无底部Tab，进入来源：我的页面 |
| 二级页面 | `/settings` | 系统设置 | 全屏，无底部Tab，进入来源：我的页面 |

> **二级页面导航规范（统一）**：
> - 进入方式：以全屏模态方式进入，不显示底部Tab导航栏
> - 故障日志：主要从"我的"页面进入，也支持从首页异常报警条点击进入
> - 维保计划/维保记录/系统设置：从"我的"页面进入
> - 返回逻辑：通过顶部导航栏的 X 按钮关闭，使用 `context.pop()` 返回上一级页面
> - 奶厅切换：二级页面顶部导航栏支持奶厅切换下拉

## 12. 消息提醒方案

### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    消息提醒架构                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   后端推送/触发源                                        │
│        │                                                │
│        ▼                                                │
│   ┌─────────────┐      ┌─────────────┐                 │
│   │  系统通知   │      │  应用内通知  │                 │
│   │  (推送)    │      │  (横幅/弹窗) │                 │
│   └──────┬──────┘      └──────┬──────┘                 │
│          │                    │                        │
│          ▼                    ▼                        │
│   ┌─────────────┐      ┌─────────────┐                 │
│   │  消息服务   │      │  消息服务   │                 │
│   │ (LocalPush) │      │(InAppBanner)│                 │
│   └─────────────┘      └─────────────┘                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 提醒类型与实现

| 提醒类型 | 实现方案 | 使用场景 |
|---------|---------|---------|
| **系统推送** | flutter_local_notifications | 维保到期、设备异常告警 |
| **应用内横幅** | Overlay 弹窗，3秒自动消失 | 操作反馈、警告提示 |
| **应用内弹窗** | Modal 对话框 | 重要确认、错误详情 |

### 消息服务设计

```dart
// 消息服务接口
abstract class MessageService {
  // 发送系统推送通知
  Future<void> sendPushNotification(String title, String body);

  // 发送应用内横幅
  Future<void> showInAppBanner(String message, MessageType type);

  // 发送应用内弹窗
  Future<void> showInAppDialog(String title, String message);
}

// 消息类型
enum MessageType {
  success,   // 成功 - 绿色
  warning,   // 警告 - 橙色
  error,     // 错误 - 红色
  info,      // 信息 - 蓝色
}
```

### 使用场景

| 场景 | 提醒类型 | 说明 |
|-----|---------|-----|
| 维保任务即将到期 | 系统推送 + 应用内弹窗 | 提前提醒用户 |
| 设备异常告警 | 系统推送 + 应用内横幅 | 实时告警 |
| 操作成功反馈 | 应用内横幅 | 非阻断式反馈 |
| 数据加载失败 | 应用内弹窗 | 需要用户确认 |

### 架构设计

```

│                    异常处理架构                           │
├─────────────────────────────────────────────────────────┤
│                                                         │

│   │           Exception Handler              │          │

│   │  │ 网络异常  │ 数据异常  │ UI异常   │  │          │
│   │  └───────────┴───────────┴───────────┘  │          │
│   └──────────────────┬──────────────────────┘          │
│                      │                                   │
│         ┌────────────┼────────────┐                     │
│         ▼            ▼            ▼                    │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐               │
│   │ 本地日志 │ │ 错误上报 │ │ 用户提示 │               │
│   │ (Hive)   │ │ (后端)   │ │ (UI)     │               │
│   └──────────┘ └──────────┘ └──────────┘               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 处理层级

| 处理层级 | 实现方案 | 说明 |
|---------|---------|-----|
| **用户提示** | 全局 ErrorWidget + Toast | 友好错误提示，不闪退 |
| **本地日志** | Hive 日志库 | 离线存储，调试查看 |
| **后端上报** | 错误收集 API | 生产环境错误追踪 |

### 全局异常捕获

```dart
// main.dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // 全局异常捕获
  FlutterError.onError = (details) {
    // 1. 本地日志记录
    ErrorLogger.log(details.exceptionAsString());

    // 2. 上报到后端
    ErrorReporter.report(details);

    // 3. 展示错误页面
    ErrorWidget.builder(details);
  };

  // 异步异常捕获
  PlatformExceptionsPlugin.instance.setHandler((exception) {
    ErrorLogger.log(exception.toString());
    ErrorReporter.report(exception);
  });

  runApp(const ProviderScope(child: MyApp()));
}
```

### 错误日志模型

```dart
// 错误日志实体
class ErrorLog {
  final String id;
  final String message;           // 错误信息
  final String stackTrace;       // 堆栈信息
  final String deviceInfo;        // 设备信息
  final DateTime timestamp;       // 发生时间
  final String page;             // 发生页面
  final String userId;           // 用户ID（脱敏）
}

// 错误上报服务
class ErrorReporter {
  Future<void> report(FlutterErrorDetails details) async {
    // 发送到后端错误收集 API
  }
}
```

### 各层异常处理策略

| 层级 | 异常类型 | 处理方式 |
|-----|---------|---------|
| **网络层** | 网络断开、超时 | 重试机制 + Toast 提示 |
| **数据层** | 数据解析、格式错误 | 日志记录 + 友好提示 |
| **业务层** | 登录失败、权限不足 | 跳转登录/弹窗提示 |
| **UI层** | 渲染异常、组件崩溃 | 降级显示 + 日志记录 |
