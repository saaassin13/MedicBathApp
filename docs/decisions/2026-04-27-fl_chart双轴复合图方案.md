# 决策：fl_chart 双轴复合图实现方案

**日期**：2026-04-27
**问题**：数据页面图表需要同时显示柱状图（药浴牛数）和折线图（乳头识别率）

---

## 背景

需要实现一个双轴复合图：
- 柱状图：蓝色，显示药浴牛数，对应左Y轴
- 折线图：橙色，显示乳头识别率，对应右Y轴
- 左右Y轴需要刻度对齐

---

## 尝试过的方案

### 方案A：Stack 叠加 BarChart + LineChart

```dart
Stack(
  children: [
    SizedBox.expand(child: BarChart(barData)),
    SizedBox.expand(child: LineChart(lineData)),
  ],
)
```

**问题**：
- BarChart 的 x 轴位置映射（`BarChartAlignment.spaceAround`）与 LineChart 的线性 x 坐标不一致
- 柱状图和折线图的 x 轴位置无法对齐
- 即使设置相同的 `maxY`，两个图表的 Y 轴刻度映射仍然不一致
- fl_chart 会根据数据自动调整轴范围，导致刻度错位

### 方案B：单一 LineChart 模拟柱状图 + 折线图

```dart
LineChart(
  LineChartData(
    lineBarsData: [
      // 折线图数据
      LineChartBarData(spots: lineSpots, ...),
      // 柱状图数据（用 belowBarData 填充模拟）
      LineChartBarData(spots: barSpots, belowBarData: BarAreaData(...)),
    ],
  ),
)
```

**优点**：
- 所有数据使用同一个坐标系，完全对齐
- x 轴、y 轴位置一一对应
- 刻度显示可控

**缺点**：
- 柱状图需要手动用 4 个点模拟矩形
- 需要处理 tooltip 区分不同数据类型

---

## 最终选择

**方案B：单一 LineChart**

选择理由：
1. 坐标系完全一致，避免了方案A的各种对齐问题
2. 代码更简洁，不需要处理 Stack 叠加的复杂性
3. 所有刻度、网格线、tooltip 都在同一坐标系下

### 1. Y轴刻度对齐


### 2. 柱状图模拟

```dart
// 用4个点形成矩形，belowBarData 填充蓝色
spots: [
  FlSpot(x - 0.3, 0),
  FlSpot(x - 0.3, cowCount),
  FlSpot(x + 0.3, cowCount),
  FlSpot(x + 0.3, 0),

```

### 3. 折线图延伸

```dart
// 首尾延伸超出边界，圆点用 radius=0 隐藏
spots: [
  FlSpot(-0.5, y),      // 延伸点（不显示圆点）
  FlSpot(0, y),         // 有效数据点
  // ...
  FlSpot(6.5, y),      // 延伸点（不显示圆点）
]
```

## 教训

1. **fl_chart 的 Stack 叠加无法共享坐标系**：即使使用 `SizedBox.expand` 让两个图表占据相同空间，它们的内部坐标系仍然是独立的。

2. **BarChart 的 x 轴映射不透明**：`BarChartAlignment.spaceAround` 会自动调整柱子位置，与 LineChart 的线性坐标不兼容。

3. **双轴图需要统一坐标系**：如果用两个独立的图表叠加，左右 Y 轴的刻度位置会不一致。需要从数据层面统一，而不是依赖图表库的"双轴"功能。

4. **优先使用单一图表**：在 fl_chart 中，用一个图表实现所有数据系列比叠加多个图表更可靠。
