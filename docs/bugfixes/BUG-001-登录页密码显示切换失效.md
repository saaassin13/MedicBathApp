# Bug #001：登录页密码显示切换失效

**现象**：点击密码输入框右侧的"显示密码"图标，密码依然保持隐藏状态。

**根因**：登录页面的密码输入框 `obscureText` 属性硬编码为 `true`，且 suffixIcon 使用静态 `Icon` 组件而非 `IconButton`，导致没有任何点击事件来处理切换逻辑。

**修复方案**：
1. 添加 `_obscurePassword` 状态变量（初始值 `true`）
2. 将 `obscureText` 绑定到该状态变量
3. 将 suffixIcon 改为 `IconButton`，添加 `onPressed` 回调切换状态

**验证方法**：
1. 运行应用，进入登录页面
2. 输入任意密码，确认密码显示为圆点
3. 点击密码框右侧的图标，密码应显示为明文
4. 再次点击，密码应恢复为圆点

**防止复发**：无。纯前端 UI 问题，无测试覆盖需求。
